# 接口与数据结构登记

本文件记录模块之间共享的接口、数据结构和调用约定。新增或修改共享接口时，应同步更新本文件；如果暂时不清楚具体函数名，不要猜测，先写“待从实现中补充”。

| 模块 | 主要职责 | 暴露接口 | 输入 | 输出 | 数据结构 | 配置依赖 | 备注 |
| --- | --- | --- | --- | --- | --- | --- | --- |
| 配置读取模块 | 读取 YAML 配置并校验系统配置结构 | `load_settings(config_path)`；`AppConfig`；`CloudModelConfig`；`LocalModelConfig`；`GenerationConfig`；`Settings` | `config_path: str | Path`，YAML 顶层为映射对象 | `Settings`；非映射 YAML 抛出 `ValueError`；字段校验失败抛出 Pydantic `ValidationError` | `AppConfig(project_root)`；`CloudModelConfig(provider, model, api_key_env, base_url)`；`LocalModelConfig(provider, model, endpoint)`；`GenerationConfig(temperature, max_tokens)`；`Settings(app, cloud_model, local_model, generation)` | YAML 配置包含 `app`、`cloud_model`、`local_model`、`generation` 四段 | Phase 1 已登记；模型供应商只作为配置字段，不写死在业务逻辑中 |
| 基础错误模块 | 提供项目通用异常类型和用户可读错误消息格式 | `ProjectError`; `ConfigError`; `EmptyTextError`; `ProjectNotFoundError`; `DuplicateProjectError`; `ModelConfigError`; `error_to_dict(error: ProjectError) -> dict`; `error_to_text(error: ProjectError) -> str` | 构造错误时可传 `message: str | None`、`suggestion: str | None`、`details: dict | None`、`code: str | None` | 用户可读字典或文本；字典字段为 `code`、`message`、`suggestion`，`details` 非空时才输出 | `ProjectError(code, message, suggestion, details)` 及其子类 | 无 | 独立基础模块；默认中文消息；当前不接入 FastAPI/WebUI/模型重试；后续模块可逐步替换裸 `ValueError` |
| 项目目录创建模块 | 根据项目根目录和项目 ID 创建标准项目目录 | `create_project_directories(project_root, project_id)`；`PROJECT_SUBDIRECTORIES` | `project_root: str | Path`；`project_id: str` | 返回项目目录 `Path`，并创建标准子目录 | `PROJECT_SUBDIRECTORIES = ("config", "sources", "analysis", "drafts", "revisions", "final", "logs")` | 不依赖配置对象；调用方可使用 `Settings.app.project_root` 作为项目根目录 | `project_id` 不能为空，不能为 `.` 或 `..`，不能包含 `/` 或 `\` |
| 项目索引 JSON 模块 | 用项目根目录下的人类可读 JSON 文件维护本地项目索引，支持新增、列表和按 ID 查询 | `PROJECT_INDEX_FILENAME`; `ProjectIndexRecord`; `add_project_record(project_root, record) -> ProjectIndexRecord`; `list_project_records(project_root) -> list[ProjectIndexRecord]`; `get_project_record(project_root, project_id) -> ProjectIndexRecord \| None` | `project_root: str | Path`; `ProjectIndexRecord(project_id, name, created_at=默认当前 UTC 时间)`; 查询时 `project_id: str` | 新增时返回规范化后的 `ProjectIndexRecord`; 列表返回索引中全部记录；按 ID 查询不存在返回 `None`; 重复 ID 抛出 `ValueError` | `ProjectIndexRecord(project_id: str, name: str, created_at: datetime)`; 文件结构为 `projects_index.json` 中的 `{ "projects": [...] }` | 无；调用方可使用 `Settings.app.project_root` 作为项目根目录 | 不使用 SQLite；不创建项目目录；`project_id` 不能为空，不能为 `.` / `..`，不能包含路径分隔符；新增重复时报 `project_id 已存在: ...`；索引文件内重复时报 `project_id 重复: ...`; JSON 使用 2 空格缩进并保留中文 |
| 基础文本清洗模块 | 清洗内存中的原始文本，为手动输入文本和后续网页抓取文本进入切分/分析流程前做准备 | `clean_text(raw_text: str) -> str` | `raw_text` 原始字符串 | 清洗后的字符串；空文本或清洗后为空抛出 `ValueError("text 不能为空")` | 无新增数据结构 | 无全局配置依赖 | 统一 `\r\n`、`\r`、`\n` 为 `\n`；去除首尾和行首尾空白；合并连续空白段落为单个空行；保留章节标题和非空行内部结构；不解析 HTML、不访问网络、不调用模型 |
| 文本切分模块 | 将长文本切分为适合后续分析和改写的片段 | `TextChunk`; `split_text(source_id: str, raw_text: str, *, chapter_title: str \| None = None, chunk_chars: int = 2000) -> list[TextChunk]` | `source_id` 非空字符串；`raw_text` 原始文本；可选 `chapter_title`；`chunk_chars > 0` | `list[TextChunk]`，空白文本返回空列表 | `TextChunk(source_id: str, chapter_title: str \| None, text: str, order: int)` | 无全局配置依赖；`chunk_chars` 为函数参数 | 优先按章节切分；章节过长按段落组合；单段过长按句子组合；仅单句超过限制时硬切；每个 chunk 保留 `source_id`、`chapter_title`、`order` |
| 用户需求结构化模块 | 将用户创作要求整理为结构化数据 | `StoryRequirement`; `StoryRequirement.from_text(raw_text: str) -> StoryRequirement` | 用户自由文本需求 `raw_text: str`；也可直接构造部分字段 | `StoryRequirement` 实例 | Pydantic 模型 `StoryRequirement`；字段：`raw_text`, `theme`, `worldview`, `protagonist`, `supporting_characters`, `character_relationships`, `plot_direction`, `style_preferences`, `forbidden_content`, `required_plot_points`, `ending_preference` | 无 | 当前 `from_text` 只保存 `raw_text`，不调用模型、不抽取字段；后续接入模型解析时需保持字段白名单、处理空/缺失/格式错误结果，并保持禁止内容和必须情节为列表语义 |
| 手动参考文本输入与保存模块 | 保存用户手动粘贴的参考小说正文和简单元数据 | `ManualTextInput`; `ManualSourceMetadata`; `save_manual_source(project_path, manual_input, *, created_at=None) -> ManualSourceMetadata` | `project_path: str | Path`; `manual_input: ManualTextInput(source_id, title, text)`; 可选 `created_at: datetime | None` | 返回 `ManualSourceMetadata`; 写入 `sources/{source_id}.txt` 和 `sources/{source_id}.json`; 空文本抛出 `ValueError("text 不能为空")` | `ManualTextInput(source_id: str, title: str, text: str)`; `ManualSourceMetadata(source_id: str, title: str, created_at: datetime, input_type: Literal["manual"] = "manual")`; 元数据字段为 `source_id`、`title`、`created_at`、`input_type=manual` | 无 | 不调用模型、不做文本切分、不抓取网页、不使用 SQLite; `source_id` 不能为空、不能为 `.` / `..`、不能包含路径分隔符; 保存时创建 `sources/` 目录 |
| 手动参考文本处理集成模块 | 编排手动参考文本保存、基础清洗、文本切分和切片文件落盘 | `ManualSourceProcessingResult`; `process_manual_source(project_path, manual_input, *, chunk_chars=2000, created_at=None) -> ManualSourceProcessingResult` | `project_path: str | Path`; `manual_input: ManualTextInput`; 可选 `chunk_chars: int`; 可选 `created_at: datetime | None` | 返回 `ManualSourceProcessingResult`; 写入 `sources/{source_id}.txt`、`sources/{source_id}.json`、`sources/{source_id}.chunks.json` | `ManualSourceProcessingResult(source_id: str, cleaned_text_path: Path, chunk_count: int, chunks_path: Path)`; 切片 JSON 顶层字段：`source_id`, `title`, `chunk_count`, `chunks`; `chunks[]` 字段：`source_id`, `chapter_title`, `text`, `order` | 无全局配置依赖；`chunk_chars` 直接透传给 `split_text` | 只复用 `clean_text`、`save_manual_source`、`split_text`; 不抓取网页、不调用模型、不使用 SQLite、不修改项目索引 JSON |
| 模型接口抽象层 | 统一云端模型和本地模型调用方式，避免业务逻辑写死供应商 | `ModelRequest`; `ModelResponse`; `ModelClient.generate(request: ModelRequest) -> ModelResponse`; `FakeModelClient.generate(request: ModelRequest) -> ModelResponse` | `ModelRequest(prompt, system_prompt=None, temperature=None, top_p=None, max_tokens=None, metadata=None)` | `ModelResponse(text, provider, model, finish_reason=None, usage=None, raw=None)` | `ModelRequest`、`ModelResponse` 为 Pydantic 模型；`ModelClient` 为 Protocol；`FakeModelClient` 可传入固定 `text/provider/model` 并记录 `last_request` | 无；不读取 API key；不访问网络 | FakeModelClient 用于后续业务单元测试：业务函数依赖 `ModelClient`，测试时注入 fake 并断言返回文本和 `last_request` |
| DeepSeek 云端模型客户端 | 通过 HTTP 调用 DeepSeek API，将结果转换为 `ModelResponse` | `DeepSeekClient(base_url, model, api_key_env, *, http_client=None).generate(request: ModelRequest) -> ModelResponse` | 构造参数：`base_url`（默认 `https://api.deepseek.com`）、`model`（默认 `deepseek-v4-flash`）、`api_key_env`（默认 `DEEPSEEK_API_KEY`）；可注入 `http_client: httpx.Client` 用于测试；`generate()` 接收 `ModelRequest` | `ModelResponse`（`provider="deepseek"`）；API key 缺失抛出 `ModelConfigError`；网络失败/非 2xx/空响应/JSON 解析失败抛出 `ProjectError` | 复用 `ModelRequest`、`ModelResponse`；HTTP 调用依赖 `httpx` | 从环境变量读取 API key；不读取 YAML 配置文件 | 实现 `ModelClient` 协议，业务层只依赖协议；测试使用 `httpx.MockTransport`，不访问真实网络；错误消息中文 |
| 风格分析服务 | 串联风格分析提示词构建器与模型客户端，执行风格分析并返回结果，可选保存报告 | `StyleAnalysisResult`; `analyze_style(chunks, model_client, *, analysis_focus=None, max_chunks=None, project_path=None) -> StyleAnalysisResult` | `chunks: list[TextChunk]` 非空；`model_client: ModelClient`；可选 `analysis_focus: list[str]`、`max_chunks: int`、`project_path: str | Path` | `StyleAnalysisResult(report, source_count, chunk_count)`；chunks 为空抛出 `EmptyTextError`；模型返回空文本抛出 `ProjectError(code="empty_response")` | `StyleAnalysisResult(report: str, source_count: int, chunk_count: int)` 为 frozen dataclass；报告保存到 `{project_path}/analysis/style_analysis_report.md` | 无；`model_client` 由调用方注入 | 业务层只依赖 `ModelClient` 协议；测试使用 `FakeModelClient`，不访问真实网络；不实现模型客户端、不调用 WebUI
| 初稿生成服务 | 串联初稿生成提示词构建器与模型客户端，根据风格报告和用户需求生成原创小说初稿，可选保存初稿 | `DraftGenerationResult`; `generate_draft(style_report, requirement, model_client, *, chapter_plan=None, target_word_count=None, chapter_count=None, project_path=None) -> DraftGenerationResult` | `style_report: str` 非空；`requirement: StoryRequirement`；`model_client: ModelClient`；可选 `chapter_plan: str`、`target_word_count: int`、`chapter_count: int`、`project_path: str | Path` | `DraftGenerationResult(draft_text)`；style_report 为空抛出 `EmptyTextError`；模型返回空文本抛出 `ProjectError(code="empty_response")` | `DraftGenerationResult(draft_text: str)` 为 frozen dataclass；初稿保存到 `{project_path}/drafts/draft.md` | 无；`model_client` 由调用方注入 | 业务层只依赖 `ModelClient` 协议；测试使用 `FakeModelClient`，不访问真实网络；不实现模型客户端、不调用 WebUI
| 本地改写提示词构建器 | 为本地模型改写单个文本片段构建 ModelRequest，纯函数不调用模型 | `build_local_rewrite_prompt(global_summary, character_sheet, chapter_summary, current_fragment, rewrite_instruction, must_keep, forbidden_changes, *, temperature=0.8, top_p=0.9, max_tokens=None) -> ModelRequest` | `global_summary: str`、`character_sheet: str`、`chapter_summary: str`、`current_fragment: str`、`rewrite_instruction: str` 均非空；`must_keep: list[str]`、`forbidden_changes: list[str]` 可为空；可选 `temperature`、`top_p`、`max_tokens` | `ModelRequest` 含中文 system_prompt（要求保留核心剧情、按指令改写、称呼一致、附修改说明）和完整上下文 prompt（含设定摘要、角色表、章节摘要、片段、改写要求、必须保留/禁止改变列表）；任一必填字符串为空抛出 `ValueError` | 复用 `ModelRequest`；无新增数据结构 | 无全局配置依赖；`temperature` 等为函数参数 | 纯函数；不调用模型、不访问网络；只处理单个片段而非整篇小说；后续本地改写服务可导入此函数构建提示词后传给 `ModelClient.generate()` |
| Ollama 本地模型客户端 | 通过 HTTP 调用 Ollama `/api/chat` 接口，将结果转换为 `ModelResponse` | `OllamaClient(base_url, model, *, timeout=120.0, num_ctx=4096, http_client=None).generate(request: ModelRequest) -> ModelResponse` | 构造参数：`base_url`（默认 `http://localhost:11434`）、`model`（默认 `qwen3:4b`）、`timeout`（默认 120s）、`num_ctx`（默认 4096）；可注入 `http_client: httpx.Client` 用于测试；`generate()` 接收 `ModelRequest` | `ModelResponse`（`provider="ollama"`）；网络失败/非 2xx/JSON 解析失败/缺少 message 字段抛出 `ProjectError`；空 content 返回空 text | 复用 `ModelRequest`、`ModelResponse`；请求参数 `temperature`/`top_p`/`max_tokens` 映射到 Ollama `options` 对象（`max_tokens` → `num_predict`）；`num_ctx` 控制上下文窗口；HTTP 调用依赖 `httpx` | 无；不读取环境变量；不读取 YAML 配置文件 | 实现 `ModelClient` 协议，业务层只依赖协议；测试使用 `httpx.MockTransport`，不访问真实网络；错误消息中文；不需要 API key |
| 本地改写服务 | 串联本地改写提示词构建器与模型客户端，对单个文本片段执行改写并返回结果，可选保存改写稿 | `LocalRewriteResult`; `rewrite_locally(global_summary, character_sheet, chapter_summary, current_fragment, rewrite_instruction, must_keep, forbidden_changes, model_client, *, temperature=None, top_p=None, max_tokens=None, project_path=None) -> LocalRewriteResult` | `global_summary: str`、`character_sheet: str`、`chapter_summary: str`、`current_fragment: str`、`rewrite_instruction: str` 均非空；`must_keep: list[str]`、`forbidden_changes: list[str]` 可为空；`model_client: ModelClient`；可选 `temperature`、`top_p`、`max_tokens`、`project_path: str | Path` | `LocalRewriteResult(rewritten_text)`；任一必填字符串为空抛出 `EmptyTextError`；模型返回空文本抛出 `ProjectError(code="empty_response")` | `LocalRewriteResult(rewritten_text: str)` 为 frozen dataclass；改写稿保存到 `{project_path}/revisions/rewritten.md` | 无；`model_client` 由调用方注入 | 业务层只依赖 `ModelClient` 协议；测试使用 `FakeModelClient`，不访问真实网络；不实现模型客户端、不调用 WebUI |
| 最小端到端闭环编排器 | 串联清洗/切分/风格分析/初稿生成/本地改写为完整流程，保存所有产物到项目目录 | `PipelineResult`; `run_minimal_pipeline(reference_text, user_requirement_text, project_path, *, source_id="manual_input", model_client, local_client, rewrite_instruction, chunk_chars=2000) -> PipelineResult` | `reference_text: str` 非空；`user_requirement_text: str`；`project_path: str | Path`；`model_client: ModelClient`（云端）；`local_client: ModelClient`（本地）；`rewrite_instruction: str`；`chunk_chars: int` | `PipelineResult(style_report, style_source_count, style_chunk_count, requirement_raw, draft_text, rewritten_fragment, rewrite_original_fragment, project_path, style_report_path, draft_path, rewritten_path, error_log)`；参考文本为空抛出 `EmptyTextError`；各阶段空响应抛出 `ProjectError` | `PipelineResult` 为 frozen dataclass，含 12 个字段；出参落盘文件：`analysis/style_analysis_report.md`、`drafts/draft.md`、`revisions/rewritten.md`、`logs/pipeline.log` | 无；`model_client` 和 `local_client` 由调用方注入 | 业务层只依赖 `ModelClient` 协议；测试使用 `FakeModelClient`/`QueueFakeModelClient`，不访问真实网络；不实现 WebUI/CLI/URL 抓取/SQLite |
| 最小 CLI 入口 | 命令行调用 5.4 编排器，支持 fake（默认）、cloud（推荐）和 real 三种模型模式 | `main(argv: list[str] \| None = None) -> int`；`story-rewrite [-r/--reference-file] [-q/--requirement] [-p/--project-path] [--model-mode fake\|cloud\|real] [--deepseek-model] [--ollama-base-url] [--ollama-model] [--ollama-timeout] [--ollama-num-ctx] ...` | fake 模式同 6.1；cloud 模式风格分析/初稿/改写全走 DeepSeek；real 模式风格分析/初稿走 DeepSeek + 改写走 Ollama | 返回 0（成功）或 1（失败），`ProjectError` 被捕获并打印中文错误和建议 | 无新增数据结构 | 无；cloud/real 模式需要环境变量 `DEEPSEEK_API_KEY`；real 模式额外需要本地 Ollama 服务 | fake 模式不访问网络；cloud 模式全流程 DeepSeek API（推荐）；real 模式 DeepSeek + Ollama（需 4GB+ 显存、Ollama 稳定）；OllamaClient 默认 num_ctx=2048 适配 4GB 显存；pipeline 自动截断风格报告至 1200 字以控制改写 prompt 长度 |

## 更新规则

- 新增共享接口时，补充模块、输入、输出和测试入口。
- 修改已有接口时，同步说明影响范围，并更新 `docs/task_status.md` 中相关任务。
- 不确定的实现细节保持“待从实现中补充”，由负责该模块的会话核对后填写。
