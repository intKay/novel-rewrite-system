# 7.2 真实模型小样例验收计划

> **策略更新（2026-05-26）**：`cloud` 模式（全流程 DeepSeek API）为推荐验收路径。`real` 模式（DeepSeek + Ollama）仅在显存充足且 Ollama 稳定时使用。验收顺序建议：**先跑 cloud 模式**，通过后再尝试 real 模式。

## 1. 验收目的

验证真实模型调用链路可跑通，产物完整且中文可读。

- **cloud 模式**：DeepSeek API 完成风格分析、初稿生成、片段改写（推荐，无需本地模型）
- **real 模式**：DeepSeek API 完成风格分析和初稿生成，Ollama 本地模型完成片段改写

## 2. 验收前提

### 2.1 cloud 模式前提（推荐）

| 检查项 | 命令 | 预期 |
|--------|------|------|
| API key 已配置 | `if ($env:DEEPSEEK_API_KEY) { "DEEPSEEK_API_KEY 已配置" } else { "未配置" }` (PowerShell) | 只显示是否已配置，不显示 key 内容 |
| 项目虚拟环境激活 | `.venv\Scripts\activate` 或 `source .venv/bin/activate` | 提示符前缀出现 `(.venv)` |
| 包已安装 | `python -c "import novel_rewrite_system; print('OK')"` | 打印 `OK` |

### 2.2 real 模式额外前提

| 检查项 | 命令 | 预期 |
|--------|------|------|
| Ollama 已安装 | `ollama --version` | 显示版本号 |
| 模型已拉取 | `ollama list` | 列表中包含 `qwen3:4b` 或 `qwen2.5:3b` |

> 若 real 模式前提未满足，先按 [7.1 Ollama Windows 本地部署计划](7.1_ollama_windows_setup.md) 完成安装配置。

## 3. 准备测试输入

### 3.1 创建参考文本文件

创建 `test_reference.txt`，内容为一段 300-500 字的武侠风格中文小说片段：

```
夜色沉下来。她推开门，听见远处的钟声。院中无人，只有几片枯叶在青石板上翻卷。

她缓步走向庭中老槐，手指抚过粗糙树皮，心中涌起千般思绪。这座宅院她住了十年，每一块砖、每一片瓦她都熟悉，可今夜月光下的院落却显得陌生起来。

忽然，身后传来一声轻咳。她没有回头，只淡淡说道："既然来了，何必躲藏。"

树影中走出一个青衣男子，约莫三十岁上下，腰间挂着一柄长剑。他抱拳施礼："在下奉师命，特来拜会。今夜之事，还请姑娘三思。"

她终于转过身，目光清冷如霜："你家师叔的账，我还没跟他算。你倒是先来了。"夜风骤起，满院落叶盘旋而上，在两人之间形成一道无形的墙。青衣男子后退半步，手已搭上剑柄。
```

### 3.2 用户需求

直接写在 CLI 命令中，不需要额外文件：

```
创作一个武侠悬疑短篇：主角是隐居剑客，被故人之子找到，卷入十年前一桩血案。风格冷峻含蓄，对话简短有力。
```

## 4. 运行验收

### 4.1 cloud 模式（推荐）

```bash
story-rewrite --model-mode cloud \
    --reference-file test_reference.txt \
    --requirement "创作一个武侠悬疑短篇：主角是隐居剑客，被故人之子找到，卷入十年前一桩血案。风格冷峻含蓄，对话简短有力。" \
    --project-path ./test-output \
    --chunk-chars 2000
```

### 4.2 real 模式（可选，需 Ollama）

```bash
story-rewrite --model-mode real \
    --reference-file test_reference.txt \
    --requirement "创作一个武侠悬疑短篇：主角是隐居剑客，被故人之子找到，卷入十年前一桩血案。风格冷峻含蓄，对话简短有力。" \
    --project-path ./test-output \
    --chunk-chars 2000
```

或短选项 + 显式模型名：

```bash
story-rewrite --model-mode real \
    -r test_reference.txt \
    -q "创作一个武侠悬疑短篇：主角是隐居剑客，被故人之子找到，卷入十年前一桩血案。风格冷峻含蓄，对话简短有力。" \
    -p ./test-output \
    --deepseek-model deepseek-v4-flash \
    --ollama-model qwen3:4b \
    --ollama-timeout 180
```

### 4.2 预期输出（stdout）

```
风格分析报告：./test-output/analysis/style_analysis_report.md
初稿：./test-output/drafts/draft.md
改写稿：./test-output/revisions/rewritten.md
日志：./test-output/logs/pipeline.log

摘要：参考文本 1 来源 / 1 切片
初稿字数：xxx
改写片段字数：xxx
```

### 4.3 预期输出文件

| 文件 | 应有内容 | 最低要求 |
|------|----------|----------|
| `test-output/analysis/style_analysis_report.md` | DeepSeek 生成的中文风格分析 | 不少于 200 字，含叙事视角/句式/情绪等分析 |
| `test-output/drafts/draft.md` | DeepSeek 生成的中文初稿 | 不少于 500 字，含角色和剧情 |
| `test-output/revisions/rewritten.md` | Ollama 改写后的片段 | 非空中文文本，含改写后的正文 |
| `test-output/logs/pipeline.log` | 管道运行日志 | 含时间、字数统计 |

## 5. 验收检查清单

逐项检查并打勾：

- [ ] **CLI 返回码为 0**（无报错退出）
- [ ] **风格分析报告有内容**：打开 `analysis/style_analysis_report.md`，确认是中文且包含叙事视角、句式特点等分析字段
- [ ] **初稿有内容**：打开 `drafts/draft.md`，确认是中文小说正文，角色和剧情一致
- [ ] **改写稿有内容**：打开 `revisions/rewritten.md`，确认是改写后的中文文本
- [ ] **中文基本可读**：各文件的中文表达自然，无明显乱码或断裂
- [ ] **无关键错误**：`logs/pipeline.log` 中无 ERROR 级别日志

## 6. 失败排查指南

若 CLI 返回非零或报错，按以下顺序排查：

### 6.1 DEEPSEEK_API_KEY 问题

**症状**：`[错误] DeepSeek API Key 未配置`

**检查**：
```bash
if ($env:DEEPSEEK_API_KEY) { "DEEPSEEK_API_KEY 已配置" } else { "未配置" }
```

**解决**：设置环境变量后重新打开终端。

### 6.2 Ollama 服务问题

**症状**：`[错误] Ollama 本地模型网络请求失败`

**检查**：
```bash
ollama list                     # 服务是否在线
```

**解决**：运行 `ollama serve` 启动服务。

### 6.3 模型未下载

**症状**：Ollama 返回 `model "qwen3:4b" not found`

**解决**：
```bash
ollama pull qwen3:4b
```

### 6.4 超时

**症状**：长时间无响应，最终超时报错

**解决**：
1. 首次运行模型加载较慢（可能 30-60 秒），等待即可
2. 增大超时：`--ollama-timeout 300`
3. 首次之后再次运行会快

### 6.5 显存不足

**症状**：Ollama 报内存/显存相关错误，或响应极慢（< 2 token/s）

**解决**：
1. 关闭其他 GPU 程序
2. 改用 `qwen2.5:3b`：`--ollama-model qwen2.5:3b`
3. 减小 `--chunk-chars 1000`

### 6.6 DeepSeek 返回空或异常

**症状**：DeepSeek 阶段报"模型返回空文本"

**解决**：
1. 确认 API key 有效且有余额
2. 确认网络可访问 `https://api.deepseek.com`
3. 用 `curl` 单独测试 API key
