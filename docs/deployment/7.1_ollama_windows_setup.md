# 7.1 Ollama Windows 本地部署计划

> **策略更新（2026-05-26）**：本地 Ollama 现为**可选后端**。MVP 第一版默认使用 `--model-mode cloud`，全流程通过 DeepSeek API 完成（风格分析、初稿生成、片段改写），无需安装 Ollama。Ollama 及本部署指南仅适用于希望使用本地模型改写的用户（`--model-mode real`）。详见 `agent.md` 第 5 节和任务 7.5。

## 1. 目标环境

| 项目 | 规格 |
|------|------|
| 操作系统 | Windows 10 / 11 |
| GPU | NVIDIA RTX 3050 Laptop，约 4GB 显存 |
| CPU | AMD Ryzen 5 5600H |
| 内存 | 16GB |
| 目标工具 | Ollama（本地模型运行器） |
| 目标模型 | Qwen 系列中文小模型 |

> **重要**：Ollama 是本地模型运行工具，负责下载、管理和推理模型。模型本身（如 qwen3:4b）需要通过 `ollama pull` 单独下载。安装 Ollama ≠ 拥有模型。

## 2. 推荐模型顺序

按适配程度从高到低：

| 优先级 | 模型 | 大小（约） | 显存占用（约） | 适用场景 |
|--------|------|--------|--------|----------|
| **首选** | `qwen3:4b` | ~2.4GB | ~3.0GB (Q4) | 中文改写首选，平衡质量与性能 |
| **备选** | `qwen2.5:3b` | ~1.8GB | ~2.5GB (Q4) | 显存紧张时使用，速度更快 |
| **谨慎尝试** | `qwen2.5:7b` | ~4.4GB | ~5.5GB+ (Q4) | 可能超出 4GB 显存，需启用部分 CPU offload |

选择原则：

- **首选 `qwen3:4b`**，最适合 RTX 3050 4GB，中文能力好，改写质量稳定
- **如果 4b 仍然显存不足**（同时开浏览器等占显存），降级为 `qwen2.5:3b`
- **不要一开始用 7b**，4GB 显存下大概率 OOM 或推理极慢
- 所有模型默认使用 Q4 量化版本，Ollama 会根据模型标签自动选择

## 3. 安装步骤

### 3.1 下载并安装 Ollama

1. 访问 Ollama 官网：https://ollama.com/download
2. 点击 "Download for Windows" 下载 Windows 安装程序（`.exe`）
3. 双击运行安装程序，按提示完成安装
4. 安装完成后，Ollama 会作为 Windows 服务自动启动（系统托盘可见图标）

### 3.2 验证安装

打开 PowerShell 或命令提示符，运行：

```powershell
ollama --version
```

预期输出：`ollama version is 0.x.x`（版本号），表示安装成功。

### 3.3 确认服务运行中

```powershell
ollama list
```

如果服务未启动，手动启动：

```powershell
# 方式一：从开始菜单启动 Ollama 应用
# 方式二：命令行启动
ollama serve
```

### 3.4 拉取模型

```powershell
# 首选模型
ollama pull qwen3:4b

# 备选模型（可选，需要时再拉）
ollama pull qwen2.5:3b
```

下载时间取决于网络，4b 模型约 2-3GB，请耐心等待。下载进度会在终端显示。

### 3.5 快速功能测试

```powershell
ollama run qwen3:4b "请用中文写一句简短自我介绍。"
```

预期输出：模型用中文回复一段自我介绍。如果返回结果且中文流畅，说明模型部署成功。

### 3.6 停止模型（释放显存）

Ollama 默认会在空闲一段时间后自动卸载模型。手动卸载：

```powershell
ollama stop qwen3:4b
```

## 4. API 验证

Ollama 默认在 `http://localhost:11434` 提供 HTTP API。

### 4.1 使用 PowerShell 测试 `/api/chat`

```powershell
$body = @{
    model = "qwen3:4b"
    messages = @(
        @{ role = "user"; content = "写一句中文问候语" }
    )
    stream = $false
} | ConvertTo-Json

Invoke-RestMethod -Uri "http://localhost:11434/api/chat" -Method Post -Body $body -ContentType "application/json"
```

预期返回 JSON，包含 `message.content` 字段。

### 4.2 使用 curl 测试（如果已安装 curl）

```bash
curl http://localhost:11434/api/chat -d '{
  "model": "qwen3:4b",
  "messages": [{"role": "user", "content": "你好"}],
  "stream": false
}'
```

### 4.3 验证 `/api/generate` 兼容接口

```powershell
$body = @{
    model = "qwen3:4b"
    prompt = "你好，请简短回复"
    stream = $false
} | ConvertTo-Json

Invoke-RestMethod -Uri "http://localhost:11434/api/generate" -Method Post -Body $body -ContentType "application/json"
```

> **本项目使用 `/api/chat` 接口**（已由 4.3 OllamaClient 封装），因为 `/api/chat` 原生支持 `system_prompt` + `user_prompt` 分离，与 `ModelRequest` 结构更匹配。

## 5. 项目配置建议

在项目配置文件（如 `config.yaml`）中设置：

```yaml
local_model:
  provider: ollama
  model: qwen3:4b
  endpoint: http://localhost:11434

generation:
  chunk_chars: 1800
  temperature: 0.8
  top_p: 0.9
```

或直接在代码中构造 `OllamaClient`：

```python
from novel_rewrite_system import OllamaClient

client = OllamaClient(
    base_url="http://localhost:11434",
    model="qwen3:4b",
    timeout=120.0,   # 本地模型首次加载较慢，超时设大
)
```

**上下文窗口建议**：

- 首选用 2048（`qwen3:4b` 默认支持 32K，但 4GB 显存下 2048 更稳定）
- 如果片段短且响应快，可尝试 4096
- 不要用默认最大值跑长文本，容易 OOM

## 6. 性能建议

### 6.1 先从短片段开始

本地改写首先用 **1000 字以内** 的片段测试。确认速度和稳定性后，再逐步提升到 1500-2000 字。

### 6.2 模型降级策略

| 现象 | 可能原因 | 解决 |
|------|------|------|
| 首次推理很慢（>30 秒） | 模型正在加载到显存 | 正常，等加载完成后续调用会快 |
| 推理速度 < 5 token/s | 显存不足触发 swap | 换 `qwen2.5:3b` |
| 返回乱码或不完整 | 上下文超限 | 减小输入片段，降低 `max_tokens` |
| 调用直接报错 | 服务未启动 | `ollama serve` 或重启 Ollama 应用 |

### 6.3 分段处理原则

不要一次输入整篇小说。每次只改写一个片段（1000-2000 字），配合：

- 全局设定摘要
- 角色表
- 当前章节摘要
- 改写要求

这个组合已经在 3.4（本地改写提示词构建器）和 5.3（本地改写服务）中实现。

### 6.4 GPU 加速确认

```powershell
ollama run qwen3:4b "test" --verbose
```

查看输出末端的 `eval_rate`（token/s），如果 > 10 token/s 说明 GPU 正常工作。如果只有 1-2 token/s，检查 NVIDIA 驱动是否正确安装，并在任务管理器中确认 GPU 被调用。

## 7. 常见问题

### 7.1 `ollama` 命令找不到

**原因**：Ollama 未安装或未加入 PATH。

**解决**：
- 重新运行 Ollama 安装程序
- 关闭并重新打开 PowerShell（PATH 需要新会话才能生效）
- 手动添加到 PATH：`%LOCALAPPDATA%\Programs\Ollama`

### 7.2 模型下载慢

**原因**：国内网络访问 Ollama 模型仓库速度不稳定。

**解决**：
- 使用代理：设置 `HTTPS_PROXY` 环境变量后重启 Ollama
- 避开高峰时段下载
- 预计 2.4GB 模型下载需 10-30 分钟

### 7.3 本地服务未启动

**症状**：运行 `ollama list` 报 `connection refused`。

**解决**：
```powershell
# 手动启动服务
ollama serve

# 如果提示端口被占用，检查是否已有 ollama 进程
Get-Process -Name "ollama*"
```

### 7.4 显存不足或速度太慢

**症状**：推理非常慢（< 2 token/s）或 CUDA OOM 错误。

**解决**：
1. 关闭其他 GPU 占用程序（浏览器、游戏等）
2. 降级到 `qwen2.5:3b`
3. 减小上下文窗口：在 `ollama run` 后用 `/set parameter num_ctx 2048`
4. 确认 NVIDIA 驱动版本 ≥ 526（旧驱动不支持部分 Qwen 模型的 CUDA 特性）

### 7.5 端口 11434 被占用

**症状**：`ollama serve` 报 `listen tcp :11434: bind: address already in use`。

**解决**：
```powershell
# 查看占用端口的进程
netstat -ano | findstr :11434

# 如果 Ollama 已在运行，不需要再启动
# 如果是其他程序占用，修改 Ollama 端口：
# 设置环境变量 OLLAMA_HOST=127.0.0.1:11435 后重启 Ollama
```

对应更新项目 base_url：`http://localhost:11435`

### 7.6 模型返回质量和速度不达预期

**症状**：中文输出不够流畅，或每次响应时间不稳定。

**检查清单**：
- [ ] 确认使用的是 `qwen3:4b` 而非默认的 `llama2` 或其他英文模型
- [ ] 运行 `ollama list` 确认模型已下载且名称正确
- [ ] 首调用较慢是正常的（模型加载到显存）
- [ ] 温度参数不宜过低（< 0.5 会导致输出重复）
