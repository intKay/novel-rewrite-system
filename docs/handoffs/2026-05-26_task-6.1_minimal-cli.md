## 暴露接口

- `main(argv: list[str] | None = None) -> int` — 可编程调用的 CLI 入口函数
- 注册命令：`story-rewrite`（`pip install -e .` 后可用）
- 可通过 `python -m novel_rewrite_system.cli` 运行

## 命令行参数

| 参数 | 简写 | 必需 | 默认值 | 说明 |
|------|------|------|--------|------|
| `--reference-file` | `-r` | 是 | — | 参考小说文本文件路径 |
| `--requirement` | `-q` | 是 | — | 用户创作要求文本 |
| `--project-path` | `-p` | 是 | — | 项目输出目录路径 |
| `--source-id` | | 否 | `manual_input` | 参考文本来源标识 |
| `--rewrite-instruction` | | 否 | `请用更细腻的描写改写当前片段` | 本地改写指令 |
| `--chunk-chars` | | 否 | `2000` | 文本切分字数上限 |

## 运行示例

```bash
story-rewrite -r ./reference.txt -q "创作一个仙侠故事" -p ./output
```

或：

```bash
python -m novel_rewrite_system.cli -r ./reference.txt -q "创作一个仙侠故事" -p ./output
```

## 输出

- stdout：打印各阶段输出文件路径和摘要（来源数、切片数、初稿字数、改写片段字数）
- 文件：`{project_path}/analysis/style_analysis_report.md`、`drafts/draft.md`、`revisions/rewritten.md`、`logs/pipeline.log`
- 返回码：0（成功）、1（失败）
