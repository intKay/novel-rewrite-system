## 运行示例

```bash
# real 模式：需要 DEEPSEEK_API_KEY 环境变量 + 本地 Ollama 运行
story-rewrite --model-mode real -r ./reference.txt -q "创作一个仙侠故事" -p ./output

# 自定义模型
story-rewrite --model-mode real --deepseek-model deepseek-chat --ollama-model qwen2.5:3b \
    -r ./reference.txt -q "创作故事" -p ./output
```

## 错误处理

| 场景 | 错误码 | 消息 |
|------|--------|------|
| DEEPSEEK_API_KEY 未设置 | 1 | `[错误] DeepSeek API Key 未配置` + 设置指引 |
| Ollama 连接失败 | 1 | `[错误] Ollama 本地模型网络请求失败` + 启动指引 |
| 模型返回空文本 | 1 | `[错误] 模型返回空文本，...失败` + 重试建议 |

所有 `ProjectError` 子类异常由 `main()` 统一捕获并打印中文错误和建议。

## 前置条件

real 模式运行前需满足：

1. 设置 DeepSeek API key 环境变量：
   ```bash
   # Linux / macOS
   export DEEPSEEK_API_KEY="sk-your-key-here"
   # Windows PowerShell
   $env:DEEPSEEK_API_KEY="sk-your-key-here"
   ```

2. 启动 Ollama 并拉取模型：
   ```bash
   ollama serve
   ollama pull qwen3:4b
   ```
