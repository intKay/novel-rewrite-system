# 交接记录：7.5 模型策略调整 —— 本地模型降级为可选

## 日期

2026-05-26

## 负责会话

OpenCode

## 背景

用户实测本地 Ollama（qwen3:4b / qwen2.5:3b）在 RTX 3050 Laptop 4GB 显存环境下稳定性较差，长 prompt 频繁触发 HTTP 502 或超时（见 7.3 验收记录）。因此第一版 MVP 不应强依赖本地模型。

## 目标

1. MVP 默认使用 DeepSeek API 完成全流程（风格分析、初稿生成、片段改写）
2. Ollama 保留为可选 local backend（`--model-mode real`）
3. 不删除 OllamaClient、不修改 ModelClient 抽象

## 三种模式行为

| 模式 | 风格分析 | 初稿生成 | 片段改写 | 需要 |
|------|---------|---------|---------|------|
| `fake` | FakeModelClient | FakeModelClient | FakeModelClient | 无 |

| `real` | DeepSeekClient | DeepSeekClient | OllamaClient | DEEPSEEK_API_KEY + Ollama |
